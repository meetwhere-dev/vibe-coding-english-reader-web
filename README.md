# Vibe Coding English Reader Web

一个面向英语阅读学习的 Web 应用。  
用户可以导入 `EPUB` 电子书，在网页中阅读英文内容，点击单词查看释义、发音，并自动沉淀到生词表中，后续再进行导出和复习。

## 项目用途

这个项目的目标不是做一个通用电子书平台，而是做一个更偏学习场景的英文阅读器。

它主要解决这几个问题：

- 把英文原版书阅读和查词放在同一个界面里
- 把“查过的词”自动沉淀成生词表，而不是读完就丢
- 给移动端和桌面端都提供相对顺手的阅读体验
- 让用户可以围绕一本书完成“导入 -> 阅读 -> 查词 -> 复习”的完整闭环

当前版本已经支持：

- 导入 `EPUB` 电子书
- 章节阅读
- 点击单词查看释义
- 浏览器朗读单词发音
- 自动加入生词表
- 生词导出
- 简单复习流程
- 阅读设置调整
- 桌面端 / 移动端的不同阅读布局

## 这个项目是怎么生成出来的

这个项目不是先写好完整设计稿再一次性开发的，而是通过和 Codex 的连续问答、连续迭代，一步一步生成出来的。

整个过程更接近下面这种协作方式：

1. 先提出一个目标
   例如：构建一个英语阅读学习网站，支持导入 `EPUB`、阅读、圈词释义、生词表、导出和复习。

2. 先让 Codex 做页面和交互设计
   不是一上来就写代码，而是先把页面结构、信息层级、交互流程梳理出来。

3. 基于反馈持续改 UI 和功能
   例如：
   - 首页应该更简洁
   - 移动端释义卡不能超出屏幕
   - 目录应该隐藏到侧边
   - 阅读时要支持更好的导航方式
   - 桌面端和移动端布局应该分开处理

4. 发现问题后继续让 Codex排查和修复
   例如：
   - `EPUB` 导入报错
   - 某些章节没有显示正文
   - 图片没有被正确提取
   - 查词弹层关闭后无法再次打开
   - Git 推送时远程配置或权限问题

5. 边做边验证
   每完成一轮修改，都会重新构建项目，并根据实际运行结果继续调整。

也就是说，这个仓库本身不仅是一个英语阅读器项目，也是一个“通过自然语言驱动开发”的实践样本：  
需求不是一次性定义完成的，而是在真实反馈中被逐步明确、逐步打磨出来的。

## 项目特点

- 以对话驱动开发，而不是先写完整 PRD
- UI 和交互通过多轮自然语言反馈持续收敛
- 功能开发和问题排查在同一个协作流中完成
- 更接近真实产品迭代，而不是一次性交付的 demo
